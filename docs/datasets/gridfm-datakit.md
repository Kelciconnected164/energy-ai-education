## What this is

[`gridfm-datakit`](https://github.com/gridfm/gridfm-datakit) is the data generation engine of the [LF Energy GridFM project](https://lfenergy.org/projects/gridfm/). It generates **synthetic, realistic power flow (PF) and optimal power flow (OPF) datasets** for training ML models — especially foundation models — on grid physics.

This is not a static downloadable dataset. It is a **library that generates datasets on demand** from standard grid reference cases (MATPOWER, PGLib).

---

## Composition

Each generated snapshot contains:

| Variable | Description |
|----------|-------------|
| Bus voltage magnitudes (`V`) | Per-bus voltage (p.u.) |
| Bus voltage angles (`θ`) | Per-bus voltage angle (rad) |
| Bus active power injections (`P`) | Net injection per bus (MW) |
| Bus reactive power injections (`Q`) | Net reactive injection (MVAR) |
| Line active power flows | Active power on each branch (MW) |
| Line reactive power flows | Reactive power on each branch (MVAR) |
| Generator dispatches | Active/reactive output per generator |
| Topology state | Which lines/transformers are in service |
| Operating limit violations | Voltage / thermal limit violations (PF only) |

**Key properties:**
- Scalable to 30,000 buses (PF) / 10,000 buses (OPF)
- Realistic load scenarios: global temporal scaling + per-bus spatial noise
- N-k topology perturbations: arbitrary line/transformer/generator outages
- Generator cost diversity for OPF generalization

---

## Collection / generation

Data is generated by running a numerical power flow or OPF solver (e.g., MATPOWER, PyPSA) across perturbed network states. The perturbations are controlled stochastically by `gridfm-datakit`.

**Reference networks:** MATPOWER cases (`case118`, `case300`, etc.) and [PGLib-OPF](https://github.com/power-grid-lib/pglib-opf) benchmark cases.

---

## EA-DX mapping

GridFM power flow snapshots are **steady-state** (not time-series), so EA-DX mapping is approximate. The closest fit:

```csv
# observations.csv — one row per (snapshot, bus, measurement)
ts,tz,entity_type,entity_id,measurement,value,unit,quality_flag,source
2024-01-01T00:00:00+00:00,,feeder,BUS-001,voltage,1.02,pu,simulated,sim
2024-01-01T00:00:00+00:00,,feeder,BUS-001,active_power,-45.3,MW,simulated,sim
```

The `entity_id` encodes the bus number (opaque, not a real grid asset).  
`quality_flag: simulated`, `source: sim` always applies.

For topology state (line outages), use `events.csv`:
```csv
ts_start,ts_end,entity_type,entity_id,event_type,severity,source
snapshot-id,snapshot-id,feeder,LINE-042,switching,info,sim
```

See the [EA-DX → GridFM notebook](../../notebooks/advanced/) (planned) for a full worked example.

## Uses

**Well-suited for:**
- Pre-training power flow surrogate models
- Benchmarking ML-OPF solvers
- Studying generalization across grid topologies
- Foundation model research (GridFM-v0 pre-training)

**Limitations:**
- Steady-state only — no dynamics, no transient stability
- Synthetic perturbations may not cover all real-world correlations
- OPF feasibility depends on solver convergence — check dataset for infeasible cases

**Should NOT be used for:**
- Real-time operational planning without extensive validation on real grid data
- Replacing, not supplementing, real utility datasets for safety-critical decisions

---

## Privacy and sensitivity

Fully synthetic — based on publicly available reference networks (MATPOWER/PGLib). No customer data, no real operational data, no CEII.

## Access

```bash
pip install gridfm-datakit

# Example: generate 1000 AC power flow snapshots for case118
python -c "
from gridfm_datakit import PowerFlowDataset
ds = PowerFlowDataset.from_matpower('case118', n_samples=1000)
ds.to_csv('case118_pf.csv')
"
```

See [gridfm-datakit documentation](https://github.com/gridfm/gridfm-datakit) for full API reference.
